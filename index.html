Ques1. Difference between “ == “ and “ === “ operators.

1. == (Loose Equality / Abstract Equality):

It compares two values for equality but performs type coercion if the values are of different types. This means it will attempt to convert one or both values to the same type before making the comparison.

5 == '5' // true, because '5' is coerced into the number 5
true == 1 // true, because true is coerced into the number 1

2. === (Strict Equality):

It compares two values for equality without performing type coercion. Both the value and the type must be the same for it to return true.

5 === '5' // false, because one is a number and the other is a string
true === 1 // false, because one is a boolean and the other is a number

Ques2.What are the differences between var, let and const?

    Var

  • Scope: Function or global-scoped.
  • Hoisting: Hoisted and initialized with undefined.
  • Reassignable: Yes, can be reassigned and redeclared in the same scope.

    • let

  • Scope: Block-scoped.
  • Hoisting: Hoisted but not initialized (Temporal Dead Zone).
  • Reassignable: Yes, but cannot be redeclared in the same scope.

    • const

  • Scope: Block-scoped.
  • Hoisting: Hoisted but not initialized (Temporal Dead Zone).
  • Reassignable: No, cannot be reassigned or redeclared, but objects/arrays can be mutated

Ques3. What is execution context

An **execution context** is the environment in which JavaScript code is executed, containing the scope, variables, functions, and the `this` value at the time of execution.

Ques4.What is creation phase and execution phase.

The **Creation Phase** is when the JavaScript engine sets up the execution context, hoisting variables and functions, and determining the value of `this`. The **Execution Phase** is when the code is executed line by line, and variables are assigned their actual values.

Ques5.What is hoisting?

**Hoisting** is JavaScript's behavior of moving variable and function declarations to the top of their scope during the creation phase, allowing them to be used before they are actually declared in the code. However, only the declarations are hoisted, not the initializations.

Ques7.How many operators do we have in JS ?

JavaScript has 8 types of operators:

  1. Arithmetic operators (e.g., +, -, *, /)
  2. Assignment operators (e.g., =, +=, -=)
  3. Comparison operators (e.g., ==, ===, >, < )
  4. Logical operators (e.g., &&, ||, !)
  5. Bitwise operators (e.g., &, |, ^)
  6. Unary operators (e.g., ++, --, typeof)
  7. Ternary operator (e.g., condition ? expr1 : expr2)
  8. Relational operators (e.g., in, instanceof)

Ques7.Explain Local Scope, Block Scope, Functional Scope and Scope Chain in javascript


1. Local Scope:
   - Refers to variables declared inside a function or block. These variables are only accessible within that specific function or block.

2. Block Scope:
   - Variables declared with `let` and `const` are **block-scoped**, meaning they are only accessible within the nearest enclosing `{}` block (e.g., inside loops, conditionals).

3. Functional Scope:
   - Variables declared inside a function are **function-scoped**, meaning they are only accessible within that function, and not outside it.

4. Scope Chain:
   - The scope chain is the hierarchy of scopes that JavaScript uses to look up variables. When a variable is accessed,
    JavaScript looks through the current scope, then the outer scopes, all the way up to the global scope.
